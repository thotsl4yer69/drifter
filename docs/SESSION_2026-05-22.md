## Real defects fixed this session

| Defect | Fix |
|---|---|
| Vivi keep-warm pinging `qwen2.5:3b` while Vivi ran `qwen2.5:1.5b` — model went cold every idle window, queries dropped to LLM-offline fallback | Aligned keep-warm to 1.5b |
| Dashboard `OLLAMA_MODEL` hardcoded to `qwen2.5:3b` (too heavy for Pi 5; runner stalled at 165% CPU). POST `/api/query` timed out 35-90s with 0 bytes received | Switched to `qwen2.5:1.5b`; prompt-side NO DATA tags + `vivi_grounding.validate()` catch the fabrication class 3b was originally chosen to prevent |
| `corpus_search()` cold-loaded `sentence-transformers` on first `/api/query` (~30s of HuggingFace HEAD roundtrips even with model cached) | `corpus.warmup()` daemon thread at dashboard boot |
| Cockpit settings overlay rendered `Object.keys(load_settings())` raw — `setup_complete` exposed as a checkbox, no validation | Schema-driven overlay (`SETTINGS_SCHEMA` + `/api/settings/schema`) with enum dropdowns, bounded numerics, server-side validation |
| 22 `setInterval` pollers in the cockpit firing constantly | Collapsed to 3; rAF-batched render with dirty flags |
| Driver name hardcoded "MAZ" in the cockpit HTML | Reads from `/opt/drifter/driver.yaml` via new `/api/driver` |
| HOLD TO TALK button POSTed `/api/query` with fake voice trigger instead of `drifter/voice/listen_now` | New `/api/voice/listen_now` POST → real MQTT publish |
| Flipper `Spoof_Detection.h5` couldn't load on Pi without full TF (~500MB) | `.h5` → `.tflite` converter + `ai-edge-litert` runtime inference |
| Fly Catcher feature spec was 7-of-10 fields (zero-padded) and sigmoid output inverted | Pulled exact 10-feature spec from upstream notebook; flipped P(spoofed)→genuine_prob |
